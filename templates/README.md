# aTee Apps 템플릿

이 폴더에는 새로운 앱을 만들 때 사용할 수 있는 템플릿 파일들이 있습니다.

## 파일 목록

### 1. `app-template.html`
새로운 앱을 만들 때 시작점으로 사용할 HTML 템플릿입니다.

**사용 방법:**
```bash
# 1. 새 앱 디렉토리 생성
mkdir -p public/새앱이름

# 2. 템플릿 복사
cp templates/app-template.html public/새앱이름/index.html

# 3. 파일 편집
# - 앱 이름 변경
# - 필요한 섹션 추가
# - 기능 구현
```

### 2. `atee-styles.css`
aTee Apps의 공통 스타일시트입니다. 모든 CSS 변수와 기본 컴포넌트 스타일이 정의되어 있습니다.

**사용 방법:**
- HTML 파일의 `<style>` 태그 안에 이 파일의 내용을 복사하거나
- 별도 CSS 파일로 링크할 수 있습니다

## 새 앱 추가 체크리스트

### 1. 프론트엔드
- [ ] `public/새앱이름/` 디렉토리 생성
- [ ] `app-template.html`을 복사하여 `index.html` 생성
- [ ] 앱 이름과 설명 수정
- [ ] 필요한 섹션과 기능 추가
- [ ] UI/UX 가이드 준수 확인

### 2. 백엔드 (필요한 경우)
- [ ] `api/새앱이름/` 디렉토리 생성
- [ ] Vercel Serverless Functions 작성
- [ ] Redis 키 네임스페이스 설정 (예: `새앱이름:`)

### 3. 랜딩 페이지 업데이트
- [ ] `public/index.html`에 새 앱 카드 추가
```html
<a href="/새앱이름" class="app-card">
    <h2>앱 이름</h2>
    <p>앱 설명...</p>
</a>
```

### 4. 테스트 및 배포
- [ ] 로컬에서 테스트
- [ ] Git에 커밋 및 푸시
- [ ] Vercel 배포 확인

## 스타일 가이드 준수 사항

새 앱을 만들 때 반드시 확인하세요:

✅ **필수 사항:**
- 파스텔 톤 색상 사용
- 모바일 줌 방지 (입력 필드 16px 이상)
- 일관된 간격과 패딩
- 버튼 최소 높이 36px
- 반응형 디자인 (모바일 우선)

❌ **금지 사항:**
- 그라디언트 배경
- 화려한 색상
- 외부 CSS 프레임워크
- 마케팅 문구

자세한 내용은 [UI_UX_GUIDE.md](../UI_UX_GUIDE.md)를 참조하세요.
